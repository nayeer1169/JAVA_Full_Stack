The 4 Pillars of OOP in Java

See how the foundation of Object-Oriented Programming is built directly into our Vehicle Rental codebase.

🧩

Abstraction

Hiding complex details and showing only essentials. We declare `Vehicle` as an `abstract class` with an abstract method:

public abstract double calculateRentalCost(int days);

Each subclass must implement its own pricing rules, hiding the complexity of calculations from the agency manager.

🔒

Encapsulation

Restricting direct access to object states. All core attributes of `Vehicle` (ID, base rate, rented status) are marked `private`.

private double baseRatePerDay;

External code must use public getters and setters, allowing us to perform validation (e.g., preventing negative rental prices).

🌿

Inheritance

Reusing code and establishing hierarchies. Specific vehicle classes extend the base `Vehicle` template using `extends`:

public class Car extends Vehicle

`Car`, `Motorcycle`, and `Truck` automatically inherit all common fields (brand, model) while adding specific properties (hasInsurance, cargoCapacity).

🎭

Polymorphism

Taking many forms. We override `calculateRentalCost` across subclasses and store everything in a base-type array:

Vehicle[] inventory = new Vehicle[10];

Iterating through the array and calling `calculateRentalCost` triggers subclass-specific methods at runtime (Dynamic Method Dispatch).
